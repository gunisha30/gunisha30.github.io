Stock Market Q/A Assistant (RAG)

  • Built a RAG pipeline to answer queries over a corpus of mixed-format stock market PDFs, including analysis reports, annual reports, AGM notices, and market news. Achieved 65% retrieval recall and 69% answer accuracy on a manually evaluated question set. Used - Amazon Titan Text Embeddings v2 (1024-dim vectors), all-MiniLM-L6-v2, ChromaDB/Milvus vector store with optional persistence to disk to avoid re-embedding on repeated runs.
  • Designed a two-stage metadata system to improve retrieval precision before semantic search. In the first stage, each PDF page is individually processed by Claude Sonnet via structured output to extract fields including company names, sectors, document type, time period, financial metrics, dividend information, and comparative analysis flags. Page-level results are aggregated into a document-level metadata record, then cached to JSON to eliminate re-extraction on subsequent runs. In the second stage, a second structured LLM call parses the user query into a QueryIntent object (companies, sectors, doc types, topic keywords), which is scored against each document's metadata using weighted rules, and the top-k scoring documents are passed as a filter to ChromaDB, narrowing semantic search to only the most relevant PDFs before retrieval.
  • Implemented a retrieval quality pipeline with multiple configurable stages. Retrieved chunks are first filtered by an LLM relevance grader that assigns a binary yes/no score to each chunk against the original query, discarding irrelevant results before answer generation. Additionally designed and integrated a cross-encoder reranking stage using cross-encoder/ms-marco-MiniLM-L6-v2, pairs each retrieved chunk with the query, scores them with a cross-encoder model, and re-orders by relevance before passing to the LLM (toggled via a flag for A/B testing). Query rewriting is applied upstream: a dedicated LLM call reformulates complex multi-part queries into 2-4 decomposed sub-queries to improve recall. The final RAG chain passes both retrieved context and the pre-filtered document metadata to the answer-generation LLM, giving it structured document-level signals (financial metrics, dividend info, company summaries) alongside raw chunk text to produce more grounded responses.

Defense Stocks Research Assistant (Multi-Agent)

  • Built a stock market assistant - a multi-agent system that routes user queries across three specialized sub-agents: a stock price agent (real-time data via yfinance), a defence sector research agent, and a RAG document research agent. The orchestrator decomposes multi-part queries into parallel tool calls where possible. Sub-agents are added as tools to the orchestrator, and are themselves individual agents with their own tool sets, forming a two-tier hierarchy. Achieved an estimated 85% tool routing accuracy (correct sub-agent selected for query type) on manual evaluation.
  • Dual-layer memory architecture with PostgreSQL, short-term conversation window + long-term cross-session recall - Implemented a two-tier memory system using LangGraph's PostgreSQL backends. Short-term memory (within-session conversation history) is handled via PostgresSaver as a checkpointer, each turn's full message state is persisted per thread_id, allowing users to resume any past session by providing its ID; LangGraph automatically reloads and appends to the full history without any manual conversation management. Long-term memory is handled via PostgresStore, at the end of every session, the complete conversation is summarized by the LLM into 2-3 sentences and stored per user_id; on subsequent sessions, all past summaries are retrieved and injected into the system prompt, giving the model persistent cross-session context (e.g., user risk preferences, stocks previously discussed). Also implemented MemorySaver (in-memory checkpointer) as a lightweight alternative for local testing. Built a separate introspection utility that reads raw checkpoint state directly from PostgreSQL and prints the full message history for any thread_id, useful for debugging state corruption without running the full graph.
  • Achieved an estimated 78% end-to-end answer accuracy on a manually curated set of 25 stock market queries spanning price lookups, sector analysis, and document-grounded questions.

Applied role/persona assignment across all prompts to activate domain-specific token distributions to improve terminology precision and reasoning depth. Enforced **structured JSON output** by defining explicit schemas with typed fields and null-fallback defaults.

Optimized RFE

  • Conducted benchmarking against scikit-learn RFE and RFECV using multiple classifiers.
  • Currently achieving same accuracy and execution time as scikit-learn RFE with fewer features.
  • Highly customisable parameters like thresholds, max_iterations etc. Eliminates need to provide n_features to the algo.
